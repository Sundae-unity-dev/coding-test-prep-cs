1. 올바른 괄호

( 와 ) 로만 이루어진 문자열 s 가 주어져요. 모든 여는 괄호가 짝이 맞게 닫히면 true, 아니면 false 를 반환하세요.

입출력 예시

s반환값
"()()"true
"(())()"true
")()("false
"(()("false
프로그래머스에서 풀기 →
여는 괄호를 만나면 +1, 닫는 괄호를 만나면 -1 하며 세어 보세요. 중간에 카운트가 음수가 되면(닫는 괄호가 더 많으면) 그 순간 false예요. 끝까지 갔을 때 0이면 모든 짝이 맞은 거예요. 스택을 쓰지 않고 정수 하나로도 풀 수 있는 게 포인트예요.
public bool solution(string s)
{
    int open = 0;
    foreach (char c in s)
    {
        if (c == '(') open++;
        else open--;
        if (open < 0) return false;   // 닫는 괄호가 더 많으면 실패
    }
    return open == 0;                  // 모두 짝이 맞아야 true
}

2. 완주하지 못한 선수

마라톤 참가자 명단 participant 와 완주자 명단 completion 이 주어져요. 완주하지 못한 단 한 명의 이름을 반환하세요. 동명이인이 있을 수 있어요.

입출력 예시

participantcompletion반환값
["leo", "kiki", "eden"]["eden", "kiki"]"leo"
["mislav", "stanko", "mislav", "ana"]["stanko", "ana", "mislav"]"mislav"
프로그래머스에서 풀기 →
동명이인 때문에 HashSet으로는 안 돼요. Dictionary에 이름별 인원수를 세어 두고, 완주자 명단을 돌며 하나씩 빼세요. 마지막에 값이 0보다 큰(=빠지지 않은) 이름이 답이에요. 두 명단을 정렬해 비교하는 방법도 있지만 해시가 더 빨라요.
public string solution(string[] participant, string[] completion)
{
    var count = new Dictionary<string, int>();
    foreach (var p in participant)
        count[p] = count.TryGetValue(p, out int c) ? c + 1 : 1;

    foreach (var c in completion)
        count[c]--;                    // 완주자는 하나씩 깎아요

    foreach (var kv in count)
        if (kv.Value > 0) return kv.Key;   // 남은 한 명이 미완주자
    return "";
}

3. 2 x n 타일링

2 x n 크기의 직사각형을 1 x 2 또는 2 x 1 타일로 채우는 방법의 수를 구하세요. 경우의 수가 커질 수 있으니 10,007로 나눈 나머지를 출력해요. (입력: 정수 n, 1 ≤ n ≤ 1,000)

입출력 예시

입력 n출력
22
45
백준에서 풀기 →
마지막 타일을 어떻게 놓는지로 경우를 나눠 보세요. 맨 끝에 세로 타일(2 x 1) 하나를 놓으면 남은 건 2 x (n-1), 가로 타일(1 x 2) 두 장을 놓으면 남은 건 2 x (n-2)예요. 그래서 dp[n] = dp[n-1] + dp[n-2], 피보나치와 같은 점화식이에요. 더하면서 나머지(% 10007)를 꼭 취하세요.
static void Main()
{
    int n = int.Parse(Console.ReadLine());
    const int MOD = 10007;
    long[] dp = new long[Math.Max(3, n + 1)];
    dp[1] = 1; dp[2] = 2;
    for (int i = 3; i <= n; i++)
        dp[i] = (dp[i - 1] + dp[i - 2]) % MOD;   // 더하면서 나머지를 취해요
    Console.WriteLine(dp[n] % MOD);
}

4. 미로 탐색

N x M 미로가 주어져요. 1 은 갈 수 있는 칸, 0 은 벽이에요. 출발점 (0, 0) 에서 도착점 (N-1, M-1) 까지 지나야 하는 최소 칸 수(출발과 도착 칸을 모두 포함)를 구하세요. 상하좌우로만 움직여요.

입출력 예시

입력 (4 x 6)출력
101111 / 101010 / 101011 / 11101115
백준에서 풀기 →
모든 이동 비용이 1인 최단 거리이니 BFS예요. 큐에서 칸을 꺼내 상하좌우 네 방향을 살피고, 격자 안이면서 길(1)이고 아직 방문하지 않은 칸이면 거리를 +1 해서 큐에 넣어요. 거리 표시는 반드시 큐에 넣는 순간 하세요. 꺼낼 때 표시하면 같은 칸이 여러 번 들어가요.
static int[] dx = { -1, 1, 0, 0 };
static int[] dy = { 0, 0, -1, 1 };

static void Main()
{
    var f = Console.ReadLine().Split();
    int n = int.Parse(f[0]), m = int.Parse(f[1]);
    int[,] maze = new int[n, m];
    for (int i = 0; i < n; i++)
    {
        string row = Console.ReadLine();
        for (int j = 0; j < m; j++) maze[i, j] = row[j] - '0';
    }

    var dist = new int[n, m];
    var q = new Queue<(int x, int y)>();
    q.Enqueue((0, 0));
    dist[0, 0] = 1;                    // 시작 칸 포함 거리 1

    while (q.Count > 0)
    {
        var (x, y) = q.Dequeue();
        for (int d = 0; d < 4; d++)
        {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx < 0 || nx >= n || ny < 0 || ny >= m) continue;
            if (maze[nx, ny] == 0 || dist[nx, ny] != 0) continue;
            dist[nx, ny] = dist[x, y] + 1;   // 큐에 넣을 때 거리 확정
            q.Enqueue((nx, ny));
        }
    }
    Console.WriteLine(dist[n - 1, m - 1]);
}

5. 가장 큰 수

0 이상의 정수 배열 numbers 가 주어져요. 이 수들을 이어 붙여 만들 수 있는 가장 큰 수를 문자열로 반환하세요. (결과가 클 수 있어 문자열로 반환해요.)

입출력 예시

numbers반환값
[6, 10, 2]"6210"
[3, 30, 34, 5, 9]"9534330"
[0, 0, 0]"0"
프로그래머스에서 풀기 →
두 수 a, b의 순서는 단순 크기나 사전순이 아니라 "이어 붙인 결과"로 정해요. 즉 b+a 와 a+b 를 문자열로 비교해서 더 큰 쪽이 앞에 오게 정렬해요. 마지막 함정 하나, 모든 수가 0이면 결과가 "000..."이 되니 맨 앞이 "0"이면 그냥 "0"을 반환하세요.
// using System; using System.Linq; 필요
public string solution(int[] numbers)
{
    string[] arr = numbers.Select(x => x.ToString()).ToArray();
    Array.Sort(arr, (a, b) => (b + a).CompareTo(a + b));  // 이어붙인 결과로 비교
    if (arr[0] == "0") return "0";     // 모두 0이면 "0" 하나로
    return string.Concat(arr);
}

6. 더 맵게

모든 음식의 스코빌 지수를 K 이상으로 만들려고 해요. 가장 맵지 않은 두 음식을 골라 가장 맵지 않은 음식 + (두 번째로 맵지 않은 음식 x 2) 로 섞는 과정을 반복해요. 모든 음식이 K 이상이 되는 최소 섞기 횟수를 반환하고, 불가능하면 -1 을 반환하세요.

입출력 예시

scovilleK반환값
[1, 2, 3, 9, 10, 12]72
프로그래머스에서 풀기 →
매번 "가장 작은 값 두 개"가 필요해요. 매번 정렬하면 느리니 최소 힙(PriorityQueue)을 쓰세요. 가장 작은 값이 K 이상이 될 때까지 두 개를 꺼내 섞은 결과를 다시 넣어요. 꺼낼 게 하나뿐인데 아직 K 미만이면 -1이에요. C#의 PriorityQueue는 기본이 최소 힙이라 그대로 쓰면 돼요.
public int solution(int[] scoville, int K)
{
    var pq = new PriorityQueue<int, int>();
    foreach (var s in scoville) pq.Enqueue(s, s);

    int count = 0;
    while (pq.Peek() < K)
    {
        if (pq.Count < 2) return -1;   // 더 섞을 게 없는데 아직 K 미만
        int a = pq.Dequeue();
        int b = pq.Dequeue();
        pq.Enqueue(a + b * 2, a + b * 2);
        count++;
    }
    return count;
}
